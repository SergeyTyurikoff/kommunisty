## Что сейчас есть

- 6 отдельных биомов с переходом через портал в конце уровня.
- На каждом биоме один обязательный босс.
- Верхние односторонние платформы: через них можно прыгать снизу и спускаться вниз по `S / ↓ / Ы`.
- Враги привязаны к своим этажам и патрульным зонам.
- Здоровье героя называется «время» (тема игры), но это обычная шкала здоровья: само не убывает, восстанавливается пикапами/аптечками/уровнями.
- **Турбо** (`C / С`) — единственный режим, который тратит здоровье: 5 секунд быстрее и сильнее, затем слабость.
- **Остановка времени** (`F / А`) замораживает врагов и пули — без затрат здоровья, ограничена откатом.
- **Чекпойнты между биомами**: при гибели `R` возвращает к началу пройденного биома с сохранённым прогрессом.
- **Волны (rush)**: перед волной на месте появляются часы с обратным отсчётом; у волны есть мини-цель (перебить всех / продержаться) с наградой.
- **Сочность**: hit-stop и тряска экрана на тяжёлых попаданиях; одних врагов при смерти разрывает на части (гибы), другие пачкают экран кровью.
- Разные типы оружия и боеприпасов: пистолет, Мосинка, пулемёт, огнемёт, газомёт, шашка, обрез.
- HUD справа сверху в стиле GTA: деньги, оружие, патроны, здоровье, XP, цель/босс.
- Подключены пользовательские PNG-ассеты через `drawImage()`.

## Механика «время как ресурс» (отменённая идея)

Изначально задумывалось, что `player.time` — это одновременно здоровье **и**
расходуемый ресурс: время убывало само, его можно было выкачивать из врагов
(`E/У`), тратить на турбо и остановку времени. От этого отказались: убывание и
выкачивание убраны из кода, осталась обычная шкала здоровья (название «время»
сохранено как тема). Заметка оставлена, чтобы помнить о замысле.

## Правило разработки

Перед каждой новой версией обязательно:

1. Проверять синтаксис:

```bash
for f in js/*.js; do node --check "$f" || exit 1; done
```

2. Запускать smoke-test:

```bash
npm test            # = node tests/playtest_smoke.js
```

3. Проверять вручную в браузере минимум:

- ходьбу на RU/EN раскладке;
- прыжок и приземление (squash);
- спуск через платформы вниз;
- магазин;
- стрельбу;
- турбо (C/С) и остановку времени (F/А);
- волну с часами и её мини-цель;
- переход между биомами и чекпойнт после смерти;
- рестарт.

## v17 notes

### Основная дорога
Нижний уровень и кислотная земля убраны. Главная дорога теперь ровная: один длинный `ground`-платформенный слой в `js/world.js`.

### Верхние платформы
Верхние платформы имеют тип `sky`. Они односторонние:
- снизу через них можно прыгать;
- сверху можно стоять;
- `S / ↓ / Ы` сбрасывает героя только через `sky`, но не через основной пол.

### Враги
Враги создаются не ближе `x = 620`, чтобы начало локации не превращалось в пробку из плесени. Каждый враг имеет:
- `patrolMin`;
- `patrolMax`;
- `floorY`.

`World.collide()` жёстко возвращает врага на его `floorY`, если он проваливается или улетает с этажа.

### Сложность биомов
В `Game.applyBiomeScaling()` у врагов растут HP, урон, XP и деньги за убийство в зависимости от `levelIndex`.

### Боссы
У боссов есть турбо:
- активируется рядом с игроком;
- временно ускоряет босса;
- отображается красным маркером;
- настраивается в `js/entities.js`.

### Ассеты
Новые PNG-ассеты лежат в:

```text
img/sliced/
```

Подключение ассетов:
- `img/sliced/units/`
- `img/sliced/weapons/`
- `img/sliced/bosses/`
- `img/sliced/ui_tiles/`

Их связывает `js/assets.js`.

### Обязательная проверка перед выдачей новой версии

for f in js/*.js; do node --check "$f" || exit 1; done
npm test
