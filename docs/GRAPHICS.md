# Графика проекта

## Где лежит графика

Реальные ассеты лежат в папке:

```text
img/
```

Сейчас это SVG-файлы. Они легче PNG, масштабируются без мыла и нормально подходят для раннего прототипа.

## Как подключается графика

Загрузка ассетов находится здесь:

```text
js/assets.js
```

Класс `KP.Assets` создаёт `Image()` для каждого SVG и потом рисует через `ctx.drawImage`.

## Почему раньше «картинки пропали»

В v9–v11 значительная часть визуала была процедурной: персонажи, стены, враги и фон рисовались кодом через canvas-команды (`fillRect`, `arc`, `ellipse`). Это не настоящие отдельные картинки, а графика внутри JavaScript. Для прототипа терпимо, для нормальной игры — унылая бухгалтерия прямоугольников.

## Как улучшать графику дальше

### Вариант 1. SVG-спрайты
Хорошо для:
- персонажей;
- UI;
- оружия;
- иконок;
- дверей;
- магазинов.

Плюсы:
- мало весят;
- легко редактировать;
- хорошо масштабируются.

Минусы:
- не лучший вариант для сложной живописной графики.

### Вариант 2. PNG-спрайты
Хорошо для:
- пиксель-арта;
- персонажей с анимациями;
- тайлсетов;
- эффектов.

Рекомендуемая структура:

```text
img/
  sprites/
    hero/
      idle.png
      run_01.png
      run_02.png
      shoot_mosin.png
    enemies/
      zombie_idle.png
      zombie_run.png
      tank.png
  tilesets/
    forest.png
    winter.png
    desert.png
  backgrounds/
    forest_bg.png
    winter_bg.png
```

### Вариант 3. Спрайт-листы
Лучший вариант для анимаций:

```text
img/spritesheets/hero.png
img/spritesheets/enemies.png
```

Потом в JS выбирается кадр по координатам.

### Вариант 4. Сгенерировать картинки
Можно:
- сгенерировать концепты через image generation;
- вручную нарезать их на PNG;
- подключить как `img/*.png`;
- заменить текущие SVG.

Важно: если референсы лежат в `.7z`, среда может не распаковать архив. Надёжнее присылать `.zip` или отдельные PNG/JPG.

## v13: подключённые референсы

В `img/references/` лежат все присланные изображения. В игру напрямую подключены:

- `img/backgrounds/forest.jpg` — лесной биом;
- `img/backgrounds/winter.jpg` — зимний биом;
- `img/backgrounds/desert.jpg` — пустынный биом;
- `img/weapons/mosin_pixel.png` — Мосинка поверх героя;
- `img/weapons/pistol_pixel.png` — пистолет;
- `img/weapons/shotgun_pixel.png` — обрез;
- `img/weapons/smg_pixel.png` — пулемёт;
- `img/weapons/flamethrower_pixel.png` — огнемёт;
- `img/weapons/sabre_pixel.png` — шашка.

Следующий нормальный шаг: делать не одиночные спрайты, а спрайт-листы с анимациями `idle/run/jump/shoot/death` для героя и основных врагов. Сейчас это всё ещё MVP, а не пиксельная Третьяковка, к сожалению для глаз.
